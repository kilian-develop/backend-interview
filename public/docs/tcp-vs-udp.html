TCP — 신뢰성 보장 메커니즘
🌊
흐름 제어 (Flow Control)
수신측이 처리 가능한 속도에 맞춰 송신 속도를 조절합니다. 슬라이딩 윈도우를 사용해 수신 버퍼 오버플로를 방지합니다.
🚦
혼잡 제어 (Congestion Control)
네트워크 혼잡 상황을 감지하여 전송량을 줄입니다. Slow Start, AIMD, Fast Retransmit 등의 알고리즘을 사용합니다.
🔁
오류 제어 (Error Control)
패킷 손실 시 재전송 요청(ARQ)을 통해 데이터를 다시 받습니다. 시퀀스 번호로 순서를 복원합니다.
🔢
시퀀스 번호 (Sequence Number)
각 세그먼트에 번호를 붙여 순서를 보장합니다. 수신 측은 ACK 번호로 어디까지 받았는지 확인 응답을 보냅니다.
UDP — 데이터그램 단위 전송
각 패킷은 독립적으로 전송됩니다. 순서도 없고, 재전송도 없습니다.

✨ Advanced — HTTP/3 & QUIC
UDP + 신뢰성 = QUIC
UDP는 프로토콜 자체에 신뢰성이 없을 뿐, 개발자가 애플리케이션 레벨에서 직접 신뢰성을 구현할 수 있습니다.
Google이 만든 QUIC은 UDP 위에 신뢰성·보안·멀티플렉싱을 구현하여 HTTP/3의 기반이 되었습니다.
HTTP/3 응용 계층
QUIC (신뢰성 + TLS 내장) 전송 계층
UDP 전송 계층
IP 네트워크 계층
🚀 0-RTT / 1-RTT 연결

TCP+TLS는 연결에 2~3 RTT 필요. QUIC은 첫 연결도 1 RTT, 재연결은 0 RTT로 가능합니다.

📦 HOL Blocking 해결

TCP는 하나의 패킷 손실이 전체를 블로킹. QUIC은 스트림 단위로 독립 처리하여 이 문제를 해결합니다.

🔒 TLS 1.3 내장

보안이 프로토콜에 내장되어 있어 별도 TLS Handshake 없이 암호화 통신이 가능합니다.

📱 연결 이동 (Migration)

Wi-Fi → LTE 전환 시 TCP는 재연결 필요. QUIC은 Connection ID로 끊김 없이 유지합니다.
